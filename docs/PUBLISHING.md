# PUBLISHING - 公開ロードマップ（Chrome ウェブストア＋OAuth 審査）

公開には**2つの審査ルート**があり、両方そろって初めて「誰でもインストールしてログインだけで使える」状態になる。さらに **OAuth スコープの選択**が審査の重さを大きく左右する。

| ルート | 何のため | 関所 |
|--------|---------|------|
| A. Chrome ウェブストア | 拡張機能そのものの配布 | ストア審査（数日〜） |
| B. OAuth 同意画面の公開 | 任意の Google ユーザーが認可できるようにする | Google 検証（スコープ次第で重い） |

## 0. 最初の分岐：スコープ戦略（最重要）

`DISTRIBUTION.md` のとおり、公開時の B ルートの重さがここで決まる。

| スコープ | 公開時の検証 | 実装 |
|---------|------------|------|
| `drive.readonly`（現状・restricted） | **重い**：検証＋**年次の第三者セキュリティ評価（CASA）**（費用・期間大） | 変更不要 |
| `drive.file`（推奨） | **軽い**：CASA 不要 | **Google Picker でフォルダ選択**に作り替え（MV3 対応の検討要） |

→ 一般公開を本気で目指すなら `drive.file` 化が現実的。社内/少人数ならテスト中のまま、または readonly で検証を通す判断もあり。

## 1. 事前準備（スコープに関わらず必要・一部準備済み）

- [x] **アイコン**（16/32/48/128）… `src/icons/`（manifest 登録済み）
- [x] **プライバシーポリシー草案** … `PRIVACY.md`（公開前に内容確定 → Web 公開して URL を取得）
- [x] **パッケージ化** … `npm run package` で `drive-web-previewer.zip` を生成（`zip` コマンド前提＝macOS/Linux。Windows では別途 zip 手段が必要）
- [ ] **ストア掲載素材**: スクリーンショット（1280x800 等）、説明文、カテゴリ、言語
- [ ] **アプリのホームページ URL**（GitHub リポジトリ等で可）
- [ ] **連絡先メール**（同意画面・ストアで使用）

## 2. Chrome ウェブストア（ルート A）

1. [Chrome Web Store Developer Dashboard](https://chrome.google.com/webstore/devconsole) に登録（**初回のみ $5 の登録料**）
2. `npm run package` で作った zip をアップロード
3. 掲載情報を入力（説明・スクリーンショット・アイコン・プライバシーポリシー URL・権限の根拠）
4. **公開範囲**を選択：
   - 「限定公開（自分/特定ユーザー）」→ 社内テストに便利（ストア審査は軽め）
   - 「一般公開」→ 通常のストア審査
5. 審査に提出 → 通れば公開。公開後に**永続的な拡張機能 ID** が確定する

> 公開版 ID が確定したら、その ID に対応する OAuth クライアント（`SETUP_OAUTH.md`）と、必要なら manifest の `key` を整える。ローカル開発 ID と公開 ID が異なる場合は OAuth クライアントを公開 ID 用に用意する。

## 3. OAuth 同意画面の公開／検証（ルート B）

1. Google Auth Platform で対象プロジェクトを開く
2. **「アプリを公開」**（テスト中 → 本番）
3. 検証フローに従って提出：
   - プライバシーポリシー URL・アプリのホームページ・ドメイン所有権確認
   - スコープ利用の正当性説明・**デモ動画**
   - **`drive.readonly` の場合は年次セキュリティ評価（CASA）**
4. 検証完了まで「未確認アプリ」警告・利用制限が残る

> 100 ユーザー以内・身内利用なら「テスト中」のままでも運用可能（検証不要）。一般公開時のみ B の検証が要る。

## 4. 公開後の運用

- バージョン更新: `manifest.json` の `version` を上げ → `npm run package` → ダッシュボードで新 zip をアップロード
- プライバシーポリシー・権限に変更があればストアと同意画面も更新

## 5. 推奨する進め方（段階）

1. **限定公開でストアに出す**（自分・社内）→ ストア配布の流れを確認（OAuth はテスト中のまま）
2. スコープ戦略を確定（`drive.file` 化するか／`drive.readonly` で検証を通すか）
3. `drive.file` 化する場合は Picker 対応を実装（別タスク）
4. 一般公開＋OAuth 検証
